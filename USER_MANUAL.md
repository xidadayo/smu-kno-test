# 知识库学习与考核系统使用手册

## 1. 系统简介

本系统用于企业内部知识库学习、AI 分级内容生成、阶段学习、阶段考试、防离开监控和管理员看板管理。

当前 MVP 已包含以下角色与流程：

- 管理员：维护学员账号、上传知识库、审核 AI 生成内容、创建学习计划、查看学习与考试数据。
- 学员：通过浏览器学习知识点、完成阶段学习、参加阶段考试。
- 阅卷/管理人员：查看考试结果和违规记录。

默认登录账号：

- 管理员：`admin` / `admin123`

系统不再内置演示学员、演示知识库、演示学习记录、演示考试结果或演示违规记录。学员账号需要管理员创建或批量导入；后台考试结果和违规记录只来自真实考试提交和真实考试端行为。

## 2. 启动系统

### 2.1 本地启动

首次使用：

```bash
npm install
npm run dev
```

启动后访问：

- 前端页面：http://localhost:5173
- 后端健康检查：http://localhost:3001/api/health

### 2.2 生产模式启动

```bash
npm run build
npm start
```

生产模式默认访问：

- http://localhost:3001

### 2.3 Docker 启动

```bash
docker compose up --build
```

启动后访问：

- http://localhost:3001

## 3. 数据保存与迁移

系统默认使用 JSON 文件保存业务数据，便于迁移到其他电脑。

重要目录：

- `data/store.json`：学员、知识点、题库、计划、考试、违规、推送记录等业务数据。
- `uploads/`：上传的知识库文件。

迁移步骤：

1. 将整个项目文件夹复制到新电脑。
2. 确认 `data/` 和 `uploads/` 一并复制。
3. 在新电脑执行 `npm install`。
4. 执行 `npm run dev` 或 `docker compose up --build`。

## 4. 页面导航

登录后，系统会按角色显示不同菜单。

管理员左侧菜单包含：

- 总览：查看关键指标、学员阶段视图、业务闭环和飞书推送记录。
- 账号：创建学员账号、批量导入账号、查看账号列表。
- 知识库：上传文档，并触发模拟 AI 生成知识点和题库。
- AI 审核：审核知识点和题目，只有审核通过后才进入发布和考试流程。
- 计划：创建学习计划，配置学习对象、推送模式和完成条件。
- 监控：查看考试违规记录。
- 数据：统一维护系统数据，支持查看、编辑和删除。

学员左侧菜单包含：

- 我的学习：分页学习个人知识点，完成一个进入下一个，并保存每个知识点的学习记录。
- 我的考试：参加个人阶段考试，查看考试结果。

## 5. 管理员操作流程

### 5.1 创建学员账号

1. 进入“账号”页面。
2. 在“创建学员账号”区域填写：
   - 姓名
   - 登录账号
   - 部门
   - 岗位
   - 飞书 User ID
3. 点击“创建账号”。
4. 系统会自动为学员创建初始学习档案。

### 5.2 CSV 批量导入学员

进入“账号”页面，在“CSV 批量导入”区域输入多行数据。

格式：

```csv
姓名,账号,部门,岗位,飞书UserID
王一,SMU1003,运营部,质检专员,ou_demo
```

点击“批量导入”后，系统会批量创建学员账号。

### 5.3 上传知识库文档

1. 进入“知识库”页面。
2. 填写文档标题。
3. 选择知识库归属部门。
4. 选择文件。
5. 点击“上传并生成待审核内容”。

归属部门说明：

- `全公司`：所有部门学员都可以学习。
- 指定部门：只有该部门学员可以学习，并且阶段考试组卷也只会使用该部门和全公司知识点。

当前系统已接入 DeepSeek V4。配置 `DEEPSEEK_API_KEY` 后，上传文档会调用 DeepSeek 生成知识点和题目，并进入 AI 审核流程。

配置示例：

```bash
DEEPSEEK_API_KEY=你的DeepSeek Key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-pro
```

如果没有配置 Key，系统会自动降级为本地模拟生成，仍然可以完整演示审核、学习和考试流程。

当前对 `.txt`、`.md`、`.csv`、`.html` 文件会读取正文。对 PDF、Word、PPT 等二进制文件会生成审核占位内容，后续可接入真实文档解析服务。

### 5.4 审核 AI 生成内容

1. 进入“AI 审核”页面。
2. 在“知识点审核”中检查标题、摘要、阶段、难度和置信度。
3. 点击“通过”发布知识点。
4. 在“题库审核”中检查题目、题型、解析、难度和分值。
5. 点击“通过”发布题目。

设计原则：

- AI 生成内容必须可追溯到来源文档。
- AI 生成题库必须审核后才能用于考试。

### 5.5 创建学习计划

1. 进入“计划”页面。
2. 填写计划名称。
3. 设置学习对象，例如部门。
4. 选择推送模式：
   - 固定难度
   - 递进难度
   - 个性化难度
5. 设置完成条件。当前阶段考试触发以“本阶段所有可见知识点均已学完”为准。
6. 设置考试题目数量。
7. 设置及格分。
8. 设置考试时长。
9. 点击“保存计划”。

考试题目数量说明：

- 阶段学习完成后，系统会按照学习计划中的题目数量自动组卷。
- 如果可用题目少于设置数量，系统会使用当前可用题目生成考试。
- 每个阶段都必须学完该阶段下归属为 `全公司` 或本人部门的全部已审核知识点。

## 6. 学员学习流程

1. 用管理员分配的学员账号登录。
2. 进入“我的学习”页面。
3. 查看当前知识点。
4. 点击“开始学习”。
5. 阅读知识点摘要和来源位置。
6. 点击“完成本知识点，学习下一个”。
7. 系统保存该知识点的学习记录，并自动切换到下一个知识点。
8. 当前阶段全部知识点完成后，系统生成阶段考试。

系统会记录：

- 学习进度
- 有效学习时长
- 当前阅读位置
- 当前阶段和难度
- 每个学员已完成的知识点
- 每个知识点的开始时间和完成时间

当本阶段所有可见知识点都完成后，系统会自动生成阶段考试任务，并写入推送记录。

学员只能看到：

- 归属为 `全公司` 的知识点。
- 归属为本人部门的知识点。

## 7. 考试流程

1. 用学员账号登录。
2. 进入“我的考试”页面。
3. 查看考试信息：
   - 阶段
   - 考试时长
   - 及格分
   - 当前状态
4. 选择答案。
5. 点击“提交考试”。
6. 系统自动评分并更新考试状态。

当前 MVP 支持客观题自动评分。

考试流转规则：

- 考试未通过：系统会清空本阶段学习完成状态，重新推送本阶段学习计划；学员需要重新学完本阶段全部知识点后再考试。
- 考试通过：系统会自动推送学习计划中的下一阶段；如果已经是最后阶段，则记录计划完成。
- 考试结果只来自真实提交，不再使用演示考试数据。

## 8. 防离开监控

考试端会记录以下行为：

- 切换标签页或考试页进入后台
- 浏览器窗口失焦
- 复制考试内容
- 打开右键菜单

违规记录会进入“监控”页面。

违规记录只来自真实考试页面行为，例如切屏、窗口失焦、复制或右键操作；系统不再内置演示违规数据。

建议规则：

- 离开考试界面 1 次：警告。
- 离开考试界面 2 次：记录违规。
- 离开考试界面 3 次：可自动交卷。
- 单次离开超过限定时间：可自动交卷。

浏览器无法绝对禁止用户离开页面，但可以检测、记录、警告和触发自动交卷。

## 9. 管理员看板

进入“总览”页面可查看：

- 总学员数
- 学习完成人数
- 待考试人数
- 通过率
- 违规记录数
- 学员阶段视图
- 业务闭环状态
- 飞书推送记录

学员阶段视图包含：

- 姓名
- 部门
- 当前阶段
- 当前难度
- 学习进度
- 考试状态
- 违规次数

进入“监控”页面可查看每次考试结果：

- 学员
- 考试阶段
- 题数
- 正确数
- 得分
- 结果
- 用时
- 提交时间

## 10. 数据增删改查

管理员进入“数据”页面后，可以按集合维护系统数据。

支持维护的数据集合：

- 账号
- 知识库文档
- 知识点
- 题库
- 学习计划
- 学习进度
- 学习记录
- 考试任务
- 考试提交
- 违规记录
- 推送记录
- 审计日志

操作方式：

1. 点击集合标签切换数据类型。
2. 点击“编辑”查看该条数据的 JSON。
3. 修改字段后点击“保存修改”。
4. 点击“删除”可删除该条记录。

删除说明：

- 删除学员会同步清理该学员的学习进度、考试任务、考试提交、违规记录和推送记录。
- 删除知识库文档会同步删除其生成的知识点和题目。
- 删除知识点会同步删除对应题目。
- 删除学习计划会同步清理对应学习进度和考试任务。
- 默认管理员账号不可删除。

## 11. 飞书推送说明

当前 MVP 默认生成模拟推送记录，不会真实发送飞书消息。

后续如需接入真实飞书推送，可在 `.env` 中配置：

```bash
FEISHU_WEBHOOK_URL=你的飞书机器人Webhook
```

然后在后端扩展推送发送逻辑即可。

## 12. 常见问题

### 11.1 页面打不开

检查开发服务是否启动：

```bash
npm run dev
```

再访问：

```text
http://localhost:5173
```

### 11.2 API 无响应

访问健康检查：

```text
http://localhost:3001/api/health
```

如果无响应，重新启动服务。

### 11.3 换电脑后数据丢失

确认迁移时复制了以下内容：

- `data/store.json`
- `uploads/`

### 11.4 上传 Word 或 PDF 后没有真实解析内容

当前 MVP 对二进制文档生成审核占位内容。若要实现真实解析，需要接入 PDF、Word、PPT 解析服务。

## 13. 系统验证

每次修改或部署后建议执行：

```bash
npm run check
```

该命令会：

1. 构建前端。
2. 启动后端冒烟测试。
3. 检查核心 API 是否正常。

如果显示 `smoke test passed`，说明基础功能验证通过。
